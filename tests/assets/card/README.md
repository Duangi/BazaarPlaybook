# ORB 匹配测试说明

## 测试图片准备

诊断程序会自动寻找测试图片来评估 ORB 匹配性能。有两种方式：

### 方式 1：使用专用测试图片（推荐）

在 `tests/assets/card/` 目录下放置三张测试图片：
- `large_sample.png` - 大型卡牌样本
- `medium_sample.png` - 中型卡牌样本
- `small_sample.png` - 小型卡牌样本

### 方式 2：自动使用现有卡牌图片

如果没有专用测试图片，程序会自动从 `assets/images/card/` 目录中随机选择 3 张卡牌图片进行测试。

## 测试内容

对每种尺寸的卡牌图片，程序会：
1. 加载图片
2. 使用 ORB 特征匹配算法在对应尺寸的特征库中搜索
3. 记录匹配耗时
4. 显示匹配结果和得分

## 结果显示

测试完成后，会在配置完成卡片中显示：
- **大型卡牌**：匹配耗时（毫秒）
- **中型卡牌**：匹配耗时（毫秒）
- **小型卡牌**：匹配耗时（毫秒）

这样用户就能清楚知道系统识别不同尺寸卡牌需要多长时间。

## 注意事项

- 如果某个尺寸的测试图片加载失败，该尺寸会显示 0 毫秒
- 匹配时间取决于特征库大小和硬件性能
- 通常小型卡牌匹配最快，大型卡牌可能稍慢
